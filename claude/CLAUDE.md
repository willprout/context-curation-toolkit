## Handoffs & Briefings

- Kickoffs/handoffs for other sessions are thin briefings -- goal, decisions already made, paid-for discoveries, definition of done, boundaries, files to read -- never step-by-step plans. The brief skill carries the template; a project's own kickoff template overrides it.
- Always render a briefing's full text inline in chat as a copy-paste block. Never hand me only a file path.

## Plans & Pushback

- If the work involves a genuine choice of approach, present the options with tradeoffs before building; otherwise just proceed.
- If I push back twice on the same point and you're restating your position rather than bringing new evidence, say so and offer a fresh-eyes fork (fresh-eyes skill) instead of continuing to defend.

## Session Endings

- End substantive working sessions with a short plain-language report: what we set out to do, what is verifiably true now (with the evidence), open items, surprises.

## Verification

- Never claim work is done without evidence: re-open the actual artifact (exported file, rendered page, query result, test output) and confirm the change landed. "The edit succeeded" is not verification.
